## Release Notes Policy

Future releases should group changes under:

- `Added` for new features.
- `Changed` for behavior or API changes.
- `Deprecated` for functionality scheduled for removal.
- `Removed` for removed functionality.
- `Fixed` for bug fixes.
- `Security` for vulnerability fixes or hardening work.
